# 主控单元 (0001)

ESP32-S3 N16R8，负责环境感知、本地交互、云端网关中继。

## GPIO 分配

| GPIO | 用途 | 状态 |
|------|------|------|
| 1 | OLED SDA | 已使用 |
| 2 | OLED SCL | 已使用 |
| 4 | DHT22 数据 | 已使用 |
| 11 | 开门按键（上拉接地） | 已使用 |
| 15 | LED RED | 预留 |
| 16 | LED YELLOW | 预留 |
| 17 | LED GREEN | 预留 |
| 21 | 喇叭 I2S BCLK | 已使用 |
| 47 | 喇叭 I2S LRC | 已使用 |
| 20 | 喇叭 I2S DIN | 已使用 |

## 功能 PRD

### 1. 温湿度采集（env.status）

| 项目 | 说明 |
|------|------|
| 传感器 | DHT22，GPIO 4 |
| 采集周期 | 每 30 秒 |
| 上报条件 | 首次上报，或温湿度变化超阈值（±0.5°C / ±2%） |
| 上报主题 | `vex/0001/status`（retained） |
| 消息格式 | `{"type":"env.status", "temp":25.3, "humidity":60}` |
| 响应指令 | `env.query` → 立即采集并上报；`status.get` → 同上 |

### 2. 本地开门按键

| 项目 | 说明 |
|------|------|
| 按键 | GPIO 11，上拉接地，按下 LOW |
| 消抖 | 200ms |
| 动作 | 向门禁 `vex/0201/cmd` 发送 `{"type":"door.unlock","duration":5}` |
| 事件 | 上报 `vex/0001/event`：`{"type":"door.open"}` |
| 提示音 | 880Hz 方波，150ms |
| 离线行为 | 门禁离线时按键仍可触发提示音（确认已按下） |

### 3. 喇叭提示音

通过 I2S 驱动 MAX98357A 播放方波提示音，无需外部音频文件。

| 事件来源 | 触发条件 | 频率 | 时长 |
|----------|----------|------|------|
| 本地开门按键 | 按下时 | 880 Hz | 150 ms |
| 门铃事件 `door.bell` | 订阅到时 | 1000 Hz | 200 ms |
| 刷卡事件 `door.card` | 订阅到时 | 880 Hz | 100 ms |

### 4. OLED 状态屏幕

| 项目 | 说明 |
|------|------|
| 屏幕 | SSD1306 128x32，I2C（SDA=1, SCL=2） |
| 刷新频率 | 每 1 秒交替显示 |
| 第一屏 | 温湿度，格式 `25.3C 60%` |
| 第二屏 | 设备状态，格式 `灯:开 锁:定`（从本地 MQTT 缓存） |

### 5. 云端 MQTT 网关

主控作为网关，桥接本地 MQTT 与云端 MQTT。

#### 5a. 上行透传（本地 → 云端）

| 订阅（本地） | 转发至（云端） |
|-------------|--------------|
| `vex/+/status` | `cloud/vex/{unit_id}/status` |
| `vex/+/event` | `cloud/vex/{unit_id}/event` |

所有本地单元的 status 和 event 均原样转发，不做过滤。

#### 5b. 下行透传（云端 → 本地）

| 订阅（云端） | 转发至（本地） |
|-------------|--------------|
| `cloud/vex/+/cmd` | `vex/{unit_id}/cmd` |

从云端主题提取 `unit_id`，拼接本地 cmd 主题后发布。

### 6. MQTT 订阅汇总

| 主题 | 用途 |
|------|------|
| `vex/+/status` | 监听所有单元状态（透传云端 + 更新屏幕） |
| `vex/+/event` | 监听所有单元事件（透传云端 + 触发喇叭） |
| `cloud/vex/+/cmd` | 接收云端指令（转发至本地） |
| `vex/0001/cmd` | 接收发给自身的指令（env.query / status.get） |

### 7. 离线与恢复

| 场景 | 行为 |
|------|------|
| WiFi 断开 | 自动重连，本地按键仍触发提示音（喇叭不依赖网络） |
| 本地 MQTT 断开 | 自动重连，重连后重新订阅并上报初始状态 |
| 云端 MQTT 断开 | 本地控制不受影响，云端指令暂不可达 |
| 云端恢复 | 下次重连成功后自动恢复透传，无需手动干预 |
